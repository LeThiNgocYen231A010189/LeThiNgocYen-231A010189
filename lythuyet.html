Câu 1 (10 điểm): Sự khác biệt giữa position: absolute và position: relative trong CSS

position: relative

Phần tử vẫn nằm ở vị trí bình thường trong trang. Khi dùng top, left thì nó dịch chuyển so với chính vị trí ban đầu của nó. Các phần tử xung quanh không bị ảnh hưởng.

.chu-de {
  position: relative;
  top: 10px;
}

position: absolute

Phần tử bị tách ra khỏi luồng trang. Nó định vị dựa vào phần tử cha gần nhất có position: relative. Nếu không có thì định vị theo cửa sổ trình duyệt.

/* Ví dụ: đặt badge góc trên phải của card */
.card {
  position: relative;
}
.badge {
  position: absolute;
  top: 0;
  right: 0;
}

Câu 2 (10 điểm): Quy trình đăng ký domain, cấu hình hosting và vai trò của DNS

Các bước đăng ký domain và cấu hình hosting

  1. Vào trang như Namecheap, GoDaddy, tìm tên miền muốn đăng ký và thanh toán.
  2. Mua gói hosting (hoặc dùng GitHub Pages miễn phí).
  3. Vào trang quản lý domain, trỏ Nameserver hoặc DNS Record (loại A) về địa chỉ IP của hosting.
  4. Upload file HTML/CSS/JS lên hosting (qua FTP hoặc File Manager).
  5. Chờ 1-24 giờ để DNS cập nhật toàn cầu.

DNS là gì?

DNS (Domain Name System) giống như danh bạ điện thoại: khi bạn gõ mywebsite.com, DNS sẽ tra ra địa chỉ IP thật (104.21.55.12) của server để trình duyệt kết nối đến.

2 công cụ deploy không cần server riêng

  • GitHub Pages: Miễn phí. Push code lên GitHub, vào Settings, chọn Pages, chọn branch và có ngay link username.github.io/tenrepo.
  • Netlify: Kéo thả thư mục là xong. Tự động có HTTPS, tên miền tùy chỉnh, và cập nhật mỗi khi push code.

Rủi ro bảo mật nếu cấu hình sai

  • Không có HTTPS: Dữ liệu gửi qua form (email, mật khẩu) có thể bị đánh cắp.
  • Lộ cấu trúc thư mục: Nếu không có file index.html, server có thể hiển thị toàn bộ các file trong thư mục.
  • Dùng mật khẩu yếu cho hosting: Kẻ tấn công có thể chiếm quyền kiểm soát.

Câu 3 (10 điểm): So sánh let, const và var về scope và hoisting

Bảng so sánh

HoistingCó (giá trị: undefined)Có (không dùng được)Có (không dùng được)
Khai báo lạiĐượcKhôngKhông
Gán lại giá trịĐượcĐượcKhông

Ví dụ lỗi thường gặp với var

// Lỗi: var trong vòng lặp
for (var i = 0; i < 3; i++) {
  setTimeout(function() {
    console.log(i); // In ra: 3, 3, 3 (sai!)
  }, 1000);
}

for (let i = 0; i < 3; i++) {
  setTimeout(function() {
    console.log(i); // In ra: 0, 1, 2 (đúng!)
  }, 1000);
}

Nguyên nhân: var có function scope nên biến i bị dùng chung. let có block scope nên mỗi vòng lặp có i riêng.

Debug bằng Chrome DevTools

  1. Nhấn F12, chọn tab Sources.
  2. Click vào số dòng cần kiểm tra để đặt Breakpoint.
  3. Chạy lại trang, code dừng tại dòng đó, xem giá trị biến ở panel bên phải.
  4. Dùng nút Step Over để chạy từng dòng một.
  5. Tab Console: gõ console.log(i) để kiểm tra nhanh.

Câu 4 (10 điểm): Event Delegation trong JavaScript

Event Delegation là gì?

Thay vì gắn sự kiện (event listener) cho từng phần tử con, ta chỉ gắn 1 listener duy nhất trên phần tử cha. Khi người dùng click vào con, sự kiện sẽ "nổi" lên cha (gọi là Event Bubbling) và cha xử lý thay.

Ví dụ thực tế: Thay vì gắn nút "Xóa" cho 100 sản phẩm, chỉ gắn 1 lần trên danh sách.

Ví dụ ứng dụng: Danh sách sản phẩm động

<ul id="danh-sach">
  <li>Sản phẩm A <button class="xoa">Xóa</button></li>
  <li>Sản phẩm B <button class="xoa">Xóa</button></li>
</ul>

<script>
document.getElementById('danh-sach').addEventListener('click', function(e) {
  if (e.target.classList.contains('xoa')) {
    e.target.parentElement.remove(); 
  }
});

function themSanPham(ten) {
  document.getElementById('danh-sach').innerHTML +=
    '<li>' + ten + ' <button class="xoa">Xóa</button></li>';
}
</script>

Lợi ích

  • Tiết kiệm bộ nhớ: Ít listener hơn, trang chạy nhanh hơn.
  • Tự động hoạt động với phần tử mới: Thêm item vào danh sách không cần gắn lại event.
  • Code gọn hơn: Xử lý tập trung ở 1 chỗ, dễ đọc, dễ sửa.
Lưu ý: Dùng e.target.closest('.class') thay vì e.target khi phần tử có nhiều thẻ con lồng nhau (tránh click nhầm).
